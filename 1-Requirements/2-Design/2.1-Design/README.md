## Опис діаграм

**1. Діаграма принципів SOLID**
Функція: Демонстрація архітектурних принципів проектування

Що описує:
- SRP - розділення відповідальностей (окремі сервіси для користувачів, постачальників, закупівель)
- OCP - можливість розширення (генератори звітів у різних форматах)
- LSP - ієрархія користувачів (адміністратор, менеджер, аналітик)
- ISP - розділення інтерфейсів (окремі контракти для аутентифікації, управління)
- DIP - залежність від абстракцій (інтерфейси репозиторіїв та сервісів)

**2. Діаграма процесу затвердження заявок**
Функція: Автоматизація workflow затвердження закупівель

Що описує:
- Багатоетапний процес затвердження заявок
- Автоматичне затвердження для малих сум (до 25,000 грн)
- Історію затвердження та відстеження статусів
- Політики безпеки та контролю доступу

**3. Діаграма сервісного шару** 
Функція: Бізнес-логіка та основні сервіси системи

Що описує:
- СервісКористувачів - реєстрація, аутентифікація, управління ролями
- СервісПостачальників - пошук, порівняння, оцінка постачальників
- СервісЗакупівель - створення заявок, затвердження, відстеження статусів
- СервісАналітики - генерація звітів, аналіз ефективності
- СервісАудиту - логування дій, відстеження активності

**4. Основна діаграма доменних класів**
Функція: Базова предметна область системи

Що описує:
- Користувач та система ролей (адміністратор, менеджер, аналітик)
- Постачальник з рейтинговою системою
- ЗаявкаНаЗакупівлю з автоматичним затвердженням
- Обладнання та специфікації
- Системи безпеки, аналітики, сповіщень
